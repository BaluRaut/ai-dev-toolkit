🧠 How It Works

Architecture Under the Hood

Not a black box. Here's exactly what happens when you run a command.

01
📦

Context Gathering

Fetches Jira REST API (ticket, AC, labels), Figma API (component tree + PNG screenshot for Vision), and OpenAPI/Swagger spec (endpoints, schemas, auth) in parallel.

02
📝

Prompt Pipeline

Context blocks merge with project rules (.aidev.yaml), auto-detected configs (tsconfig, eslint, prettier), and existing code patterns. Creates a mega-prompt matching YOUR codebase exactly.

03
🤖

Claude Generation

Claude receives structured prompt + Figma screenshot (Vision). Generates: component, service layer, TypeScript types, Jest tests, Playwright E2E, and barrel exports in one API call.

04
📄

Safe File Generation

Files parsed from fenced code blocks with path headers. Rich diff preview BEFORE anything touches disk. Confirm to write or reject. No surprise overwrites.

05
🔄

Agentic Loops

For self-healing tests and MCP: execute → read output → fix → re-execute. Each iteration tracks attempt history to prevent infinite loops. Hard cap at N retries.

06
💰

Token Tracking

Every API call intercepted: model, input/output tokens, USD cost, workflow label, timestamp. Supports 30+ models including Claude, GPT, Gemini, and Grok pricing.

🚀 Start Here

Up and Running in 3 Steps

From zero to your first AI-generated feature in under 2 minutes.

1

Install

# Clone & set up
git clone https://github.com/BaluRaut/ai-dev-toolkit.git
cd ai-dev-toolkit/claude-agent
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# Add your API key
cp .env.example .env
# Edit .env → ANTHROPIC_API_KEY=sk-ant-...
2

Try Interactive Mode

# Works instantly — no Jira/Figma keys needed
python main.py --interactive

# Paste any ticket text, screenshot, or API spec

Interactive mode lets you paste any context manually. Perfect for your first run.

3

Run Your First Workflow

# Review a file
python main.py --review src/App.tsx

# Generate unit tests
python main.py --unit-test src/components/Button.tsx

# Full workflow
python main.py --jira PROJ-123 --figma URL --swagger ./api.json

Start with --review on a file you know well. See the quality, then trust it with full workflows.

⚠️ Limitations

What You Should Know

We believe in transparency. Here's where the toolkit has boundaries.

💰

Token Costs

Each workflow uses ~$0.05–$0.30 in API tokens (Claude Sonnet). Use --costs to track exact spend.

🔎

Always Review Output

AI code should be reviewed before production. Treat it like a fast junior developer's PR — great starting point, needs human eyes.

📋

Jira Ticket Quality Matters

Vague tickets with no AC produce vague code. Write clear acceptance criteria and attach Figma links for best results.

🎨

Figma Design Fidelity

Complex auto-layout and variants work ~90% of the time. Custom animations or complex SVGs may need manual tweaking.

🔌

MCP Server Stability

MCP is a new protocol. Built-in presets (Figma, Jira, GitHub) are well-tested. Custom servers may need debugging.

🧠

Context Window Limits

Very large files (2000+ lines) may exceed Claude's context. RAG chunking handles most cases, but extremes may truncate.

🎯 Decision Guide

Which Command Should I Use?

Every scenario mapped to the right command.

🚀
Feature Development
Ticket to production-ready code
You have a Jira ticket + Figma design + Swagger spec
python main.py --jira PROJ-123 --figma URL --swagger ./api.json
The gold path. Claude reads all 3 sources → complete feature with tests.
You only have a Jira ticket
python main.py --jira PROJ-123 --stack "React 18, TypeScript, AntD 6"
Generates from ticket description and acceptance criteria alone.
No API keys set up yet
python main.py --interactive
Paste ticket text, screenshot, and endpoints manually. Zero credentials.
🔍
Code Quality & Bug Fixes
Reviews, security audits, crash fixes
You want a thorough review before raising a PR
python main.py --review src/services/paymentService.ts
Severity-graded review: bugs, security, performance, TypeScript strictness.
Your app crashed with an error
python main.py --fix "TypeError: Cannot read properties..."
Root cause analysis + patched code block.
🧪
Testing
Unit tests, E2E, auto-healing CI
A component has zero test coverage
python main.py --unit-test src/components/CheckoutForm.tsx
8–15 Jest + RTL tests covering all states, events, accessibility.
CI tests are failing after a major refactor
python main.py --heal ./ --heal-framework jest --heal-retries 5
Runs tests, reads failures, patches code, loops until green.
You need browser-level E2E tests
python main.py --e2e src/pages/Checkout.tsx --base-url http://localhost:3000
Full Playwright spec with assertions and network intercepts.
⚙️
Automation & Infrastructure
CI/CD, RAG search, MCP, costs
Your repo has no CI/CD
python main.py --ci --output ./
3 GitHub Actions: AI PR Review, Auto Tests, Quality Gate.
You need to find where something is used
python main.py --index ./ --search "JWT token refresh"
Semantic code search — beats grep for intent queries.
You want Claude to use external tools
python main.py --mcp figma --mcp-task "Extract design tokens"
Agentic loop with MCP tool orchestration. 9 presets.
🧩
Frontend & UI
A11y, styles, analytics, i18n
Your app fails a WCAG audit
python main.py --heal-a11y ./src
Scans all JSX for violations and auto-patches.
You need analytics tracking
python main.py --add-analytics ./src --app-name "My SaaS"
Full typed Amplitude SDK integration.
You need multiple languages
python main.py --add-i18n ./src --locales en,es,fr,de,ja,zh
Extracts strings, generates JSON translations, wires react-i18next.
