⚡ Get Started

Install & Configure

Clone, configure your API keys, and run your first workflow in under 2 minutes.

# 1. Clone the repository
git clone https://github.com/BaluRaut/ai-dev-toolkit.git
cd ai-dev-toolkit/claude-agent

# 2. Create virtual environment & install deps
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# 3. Copy env template
cp .env.example .env
# Required
ANTHROPIC_API_KEY=sk-ant-...

# Optional integrations
JIRA_BASE_URL=https://yourorg.atlassian.net
JIRA_EMAIL=you@company.com
JIRA_API_TOKEN=<atlassian-token>

FIGMA_ACCESS_TOKEN=figd_...

# Analytics (optional)
NEXT_PUBLIC_AMPLITUDE_API_KEY=your_amplitude_key
# Interactive menu (recommended for first-time users)
python main.py

# Full workflow: Jira + Figma + Swagger to Code
python main.py --jira PROJ-123 --figma https://figma.com/file/abc --swagger ./api.json

# Review a file
python main.py --review src/components/UserCard.tsx

# Show all session costs
python main.py --costs

Feature Development
Building new features from ticket to production-ready code
You have a Jira ticket + Figma design + Swagger spec
python main.py --jira PROJ-123 --figma URL --swagger ./api.json
The gold path. Claude reads all 3 sources and produces a complete feature: React component, service layer, Jest + RTL unit tests, Playwright E2E spec, and TypeScript types — all in one run.
You only have a Jira ticket (no Figma or API spec yet)
python main.py --jira PROJ-123 --stack "React 18, TypeScript, AntD 6"
Claude generates from the ticket description and acceptance criteria alone. Use --stack to tell it the exact tech stack when it differs from your .env defaults.
You have a Figma design + API spec but no Jira ticket
python main.py --figma https://figma.com/file/abc --swagger ./api.json
Claude analyses the visual layout and API contract then generates a matching React component with the correct data-fetching layer, loading/error states, and TypeScript interfaces.
No API keys set up yet — new hire, demo, client workshop, or CI-free environment
python main.py --interactive
Paste ticket text, screenshot path, and API endpoints manually at the prompt. Zero credentials required. Perfect for onboarding sessions or running live demos without exposing .env secrets.
You want to verify what context Claude will see before burning tokens on a full run
python main.py --jira PROJ-123
Fetches and pretty-prints the Jira ticket context string exactly as Claude would receive it. Confirm acceptance criteria and description are complete before committing to a full generation run.
You need to confirm the Figma design exports correctly before coding
python main.py --figma https://figma.com/file/abc123
Prints the component tree, colour palette, fonts, and confirms the PNG screenshot rendered correctly. If the screenshot looks wrong, check FIGMA_ACCESS_TOKEN scope — it needs read:files permission.
The generated code must skip test files (Figma-only design handoff)
python main.py --jira PROJ-123 --figma URL --swagger ./api.json --no-tests
Skips unit test + E2E generation. Useful when your team writes tests manually from acceptance criteria, or when you're generating UI-only components for a design review.

Code Quality & Bug Fixes
Pre-PR reviews, security audits, and production crash fixes
You want a thorough review of a file before raising a PR
python main.py --review src/services/paymentService.ts
Produces a severity-graded review (Critical / High / Medium / Low) covering: bugs, race conditions, security holes, TypeScript strictness, React best-practice violations, and SOLID/DRY violations — with concrete fix suggestions and code snippets.
Your app crashed with an error in the console or in CI logs
python main.py --fix "TypeError: Cannot read properties of undefined (reading 'map')"
Paste the full stack trace. Claude reasons through the call chain chain-of-thought style, identifies the root cause, and produces a patched code block with an explanation of exactly why the fix works.
A file has a bug but the CI gives a vague failure with no stack trace
python main.py --fix src/hooks/useAuth.ts
Pass a file path instead of an error string. Claude reads the code and performs a combined review + targeted fix — useful when you know which file is broken but not exactly what the error is.
A junior developer needs feedback before code review to learn from it
python main.py --review src/components/DataTable.tsx
Every issue includes an educational "why this matters" explanation alongside the fix. Great for turning AI feedback into a learning opportunity during pair sessions — not just a diff to apply blindly.
You're auditing authentication, payment, or sensitive data handling code
python main.py --review src/api/auth.ts
The review prompt explicitly checks OWASP Top-10: XSS, SQLi, SSRF, insecure deserialization, broken auth, hardcoded secrets, CORS misconfig, and missing input validation — all in one pass.
Sprint retrospective — you want to audit everything merged this week
python main.py --review src/features/billing/
Pass a directory path. The agent reviews each file and aggregates all findings by severity. Ideal for pre-release sweeps of critical modules like the billing or auth feature folder.

Testing
Unit tests, browser E2E, and auto-healing CI failures
A component or service has zero test coverage and the PM is asking about it
python main.py --unit-test src/components/CheckoutForm.tsx
Generates 8–15 Jest + RTL tests covering: renders correctly, all prop variants, user events (click/type/submit), loading/error/empty states, accessibility roles (getByRole), and a snapshot test. Files are immediately runnable — no manual wiring needed.
You need browser-level testing for a critical user journey (checkout, auth, onboarding)
python main.py --e2e src/pages/Checkout.tsx --base-url http://localhost:3000
Generates a Playwright spec: page.goto, getByRole interactions, form fills, assertions, network intercepts for API mocking, error state handling, mobile viewport test, and a playwright.config.ts ready for CI.
CI tests are failing after a major refactor and you've spent hours on it
python main.py --heal ./ --heal-framework jest --heal-retries 5
Runs the test suite, reads every failure, patches source code, and loops until all tests pass or retries are exhausted. The agent tracks each fix attempt to avoid infinite loops. Saves hours of manual debugging — especially after large dependency upgrades.
A single test file keeps failing after a library version bump
python main.py --heal ./src/components --heal-framework vitest
Scope to a subdirectory for faster loops. Supports jest, vitest, playwright, and pytest. The agent reads actual test runner output and patches source files — it never breaks tests to make them pass.
You're doing TDD — you want tests written before implementation starts
python main.py --unit-test src/services/notificationService.ts
Run on a file that has type signatures and JSDoc but no implementation. Claude generates comprehensive tests from the interface. Implement the functions until the generated test suite goes green.
You're refactoring shared code and need a regression safety net first
python main.py --unit-test src/utils/dateFormatters.ts
Generate tests covering every exported utility before touching the implementation. Run them after each change to catch regressions immediately — much faster than waiting for code review to spot broken behaviour.
You need E2E tests for an entire pages/ directory before a big release
python main.py --e2e src/pages/ --base-url https://staging.myapp.com
Pass a folder. The agent collects all page components and generates a spec file per page, covering the most critical user journey for each. Run against your staging URL to validate the full app before go-live.

Automation & Infrastructure
CI/CD pipelines, semantic codebase search, and token tracking
Your repo has no CI/CD and you need to set up pipelines fast
python main.py --ci --output ./
Generates 3 GitHub Actions workflows: AI PR Review (Claude posts review comments on every PR), Auto-Generate Tests (opens a PR with generated tests for any uncovered source files), and Quality Gate (ESLint + TSC + test coverage threshold blocks merge).
A new team member joined and needs to navigate a large unfamiliar codebase
python main.py --index ./src
Embeds the entire codebase into a local ChromaDB vector store — no external API, fully offline, persisted in .rag_index/. Takes ~2 min for a typical React app. Run once, then answer queries with --search indefinitely.
You need to find where a concept or pattern is used across hundreds of files
python main.py --index ./ --search "JWT token refresh logic"
Returns top-10 most semantically similar code chunks with file paths, line numbers, and relevance scores. Beats grep for intent-based queries — catches comments, variable names, and implementation patterns in the same search.
You want impact analysis before refactoring a shared utility or hook
python main.py --index ./ --search "usages of useUserPermissions hook"
Semantic search finds all call-sites of a pattern — even if the actual name varies slightly. Gives you full blast-radius confidence before modifying shared code that's consumed across the app.
You want to document how a complex flow works (for a wiki or PR description)
python main.py --index ./ --search "checkout payment flow from cart to confirmation"
Returns the most relevant code chunks. Say "yes" when prompted to use them as context, then ask Claude to "write a technical walkthrough of this flow for our engineering wiki".
You want to track AI token costs and justify the ROI to your manager
python main.py --costs
Prints every API call in the session: model used, input/output tokens, estimated USD cost per workflow. Run at the end of any multi-command session to see exact spend. Data is aggregated per workflow type so you can compare ROI.

Frontend & UI Agents
Components, accessibility, style migration, and design systems
Your app fails a WCAG 2.1 AA accessibility audit or a client compliance requirement
python main.py --heal-a11y ./src
Scans all .tsx/.jsx for: missing alt text, unlabelled inputs, icon buttons without aria-label, onClick on non-interactive elements (div/span), missing aria-live on dynamic content, broken focus traps on modals, and non-descriptive link text. Auto-patches every violation.
You want to preview what would change before committing a11y fixes
python main.py --heal-a11y ./src --a11y-dry-run
Prints a full diff of every proposed change without writing to disk. Review with your accessibility consultant or get stakeholder sign-off before modifying production components. Then re-run without the flag.
One specific component needs an urgent a11y fix for a client demo tomorrow
python main.py --heal-a11y ./src --a11y-file src/components/Modal.tsx
Scoped to a single file — faster and lower risk for production hotfixes. Modals, Drawers, Dialogs, and Comboboxes are the most common failing components in axe/Lighthouse accessibility audits.
You need a new reusable UI component with tests, TypeScript types, and a barrel export
python main.py --new-component PricingTable --component-desc "3-tier pricing with monthly/annual toggle" --style-lib antd
Generates 4 files: Component.tsx (ARIA-compliant), Component.test.tsx (8+ test cases covering all prop variants), Component.types.ts (full TypeScript interface), and index.ts barrel export. Uses atomic design: atom / molecule / organism / page.
Your codebase has hundreds of inline style={} objects and you can't support dark mode
python main.py --refactor-styles ./src --refactor-target styled-components
Walks every .tsx/.jsx, extracts inline style objects into Styled Components S.* namespace. Preserves dynamic/conditional styles (ternary expressions, template literals). Annotates every change with // refactored: styled-components. Safe to run incrementally per-directory.
You're migrating from plain CSS or CSS Modules to AntD 6 design system tokens
python main.py --refactor-styles ./src --refactor-target antd
Maps CSS properties to AntD 6 tokens where possible — spacing → Space/Flex, colour → ConfigProvider token, text → Typography component. Unmapped styles get wrapped in AntD's css utility. Preserves all existing behaviour.

Analytics & Internationalisation
Typed event tracking, multilingual support, RTL, and CMS-ready JSON
Your app has no analytics and you need event tracking set up from scratch
python main.py --add-analytics ./src --app-name "My SaaS" --env-prefix NEXT_PUBLIC
Scans your codebase for user interactions (onClick, onSubmit, route changes, feature flags) and generates a complete Amplitude SDK v2 integration: typed events.ts, amplitude.ts init file, useTracking hook, and barrel export — no manual event planning required.
Analytics event names are inconsistent across the team (magic strings, typos, missing properties)
python main.py --add-analytics ./src --app-name "My SaaS"
All events are typed as discriminated union constants in SCREAMING_SNAKE_CASE. TypeScript will error at compile time if you call track() with an undefined event or wrong property type — catches mistakes before they reach production data.
You need to track a full conversion funnel (page views → sign-up → checkout → activation)
python main.py --add-analytics ./src --app-name "My SaaS" --env-prefix VITE
The generated usePageTracking, useFormTracking, and useFeatureTracking hooks instrument the full user journey automatically. Drop them into your Router and form components for instant funnel data in Amplitude.
You need to launch your app in multiple languages for an international market
python main.py --add-i18n ./src --locales en,es,fr,de,ja,zh --i18n-ns translation
Extracts all hardcoded strings from JSX, generates JSON translation files for each locale, and wires up react-i18next v14: i18n.ts, TypeScript namespace (IDE autocomplete for t()), useAppTranslation hook, and AntD 6 LanguageSwitcher dropdown component.
You want t('some.key') to cause a TypeScript error if the key is missing
python main.py --add-i18n ./src --locales en --i18n-ns ui
Run English-only first to generate a fully typed i18next.d.ts namespace file. Once this is in place, any call to t() with an undefined key is a TypeScript compile error — catch missing translations at build time, not in production.
Your app needs RTL (right-to-left) layout for Arabic or Hebrew markets
python main.py --add-i18n ./src --locales en,ar
Arabic translation JSON includes RTL annotations. The LanguageSwitcher sets dir="rtl" on the document root when Arabic is selected. Your existing Flexbox / AntD Flex layout will mirror automatically with zero CSS changes.
Your translations need to feed into a headless CMS or translation management platform
python main.py --add-i18n ./src --locales en,es,fr,de --i18n-ns content
Output is standard i18next-format JSON — compatible with Phrase, Lokalise, Crowdin, and most headless CMSs. The generated key hierarchy (common.save, auth.sign_in, dashboard.empty_state) slots directly into most TMS platforms with no reformatting.

MCP — Claude + External Tools
Figma, Jira, GitHub, local files, databases — real-time agentic access
You want Claude to read and reason over your local codebase without pre-indexing
python main.py --mcp filesystem --mcp-task "Find all components still using the old Button API"
Claude opens and reads actual files using the filesystem MCP server. No RAG indexing step — works on-demand for any query. Useful for one-off investigations where you don't need a persistent vector store.
You want Claude to query Jira and produce a smart sprint summary or risk report
python main.py --mcp jira --mcp-task "List all open P1 bugs, owner, and days open. Flag any unassigned ones."
Uses uvx mcp-atlassian. Credentials auto-injected from JIRA_BASE_URL, JIRA_EMAIL, JIRA_API_TOKEN — the same ones already in your .env. Claude can run JQL searches, read issue details, comments, sprint boards, and synthesise a team-readable summary.
You want Claude to audit a Figma file for design inconsistencies or extract tokens
python main.py --mcp figma --mcp-task "List all colour and typography variables. Flag any that aren't from the approved token set."
Uses npx -y figma-mcp (official Figma MCP). FIGMA_ACCESS_TOKEN auto-injected. Claude navigates the file structure, reads component properties, variables, and dev-mode CSS — without you having to open Figma at all.
You want Claude to review GitHub PR status and surface what's blocking releases
python main.py --mcp github --mcp-task "List all PRs older than 3 days — what check or reviewer is blocking each one?"
Requires GITHUB_TOKEN in .env. Claude queries repos, pull requests, review requests, and CI check runs. Great for release managers doing standup prep or for identifying review bottlenecks at the end of a sprint.
You want Claude to query a Postgres database without writing SQL
python main.py --mcp postgres --mcp-task "Show me the 10 users with the most failed login attempts in the last 7 days"
Uses npx -y @modelcontextprotocol/server-postgres with DATABASE_URL from .env. Claude inspects the schema first, then constructs and runs the correct query, returning formatted results. No SQL knowledge needed.
You don't know what tools a specific MCP server exposes and want to explore it
python main.py --mcp jira --mcp-list
Connects to the server and pretty-prints every tool with its description and JSON Schema input parameters. No task is executed — purely exploratory. Run this first when working with an unfamiliar MCP server.
You want to connect to an internal custom MCP server (SSE or Streamable HTTP)
python main.py --mcp http://internal-tools.company.com/mcp --mcp-task "List available reports"
Supports any SSE or Streamable HTTP MCP server, not just the built-in presets. Transport type is auto-detected from the URL (SSE if path contains /sse, Streamable HTTP otherwise). Claude connects, discovers tools, and runs the agentic loop exactly as with presets.

Self-Healing Test Loop
Run tests → AI fixes failures → repeat until all pass
▾
python main.py --heal ./my-react-app --heal-framework jest --heal-retries 5
⚙️ Under the Hood
STEP 01
Run the full test suite via subprocess
Executes npx jest / vitest run / playwright test / pytest. Captures stdout, stderr, and exit code.
STEP 02
Parse failing test names and error output
Extracts structured failure objects: test name, assertion that failed, and stack trace line number.
STEP 03
Send failures + source files to Claude
Context includes: failing test, source file being tested, and all previous fix attempts to prevent infinite loops.
STEP 04
Apply Claude's fix to disk
Writes patched files. Records attempt history with timestamps and diff summaries.
STEP 05
Re-run suite → loop until green or max retries hit
Terminates on all-pass or max-retries exceeded. Prints a per-attempt healing report with pass/fail counts.
💡 Use Cases
After a major refactor — fix all broken tests in one command Dependency upgrade — new library version breaks 30 tests, heal them overnight CI pipeline gate — run heal loop automatically before merge Junior devs — let the agent fix the test red before code review

MCP Agent — Claude + Model Context Protocol
Connect any MCP server (stdio / SSE / HTTP) and run a full Claude agentic loop with its tools
▾
python main.py --mcp 'npx -y @modelcontextprotocol/server-filesystem .' --mcp-task "List all exported components in src/"
⚙️ Under the Hood
STEP 01
Parse server string into transport config
Detects transport from the string: stdio (any executable), SSE (URL ending in /sse), or Streamable HTTP (other HTTP URL). Constructs MCPServerConfig with command/args or URL accordingly.
STEP 02
Open MCP session + discover tools
Opens an async session via the mcp Python SDK. Calls list_tools() on the server — returns names, descriptions, and JSON Schema input schemas for every tool. Optionally also lists resources.
STEP 03
Convert MCP tools → Claude API tool schemas
Maps each MCPTool to Anthropic's tool format (name, description, input_schema). Server-name prefix prevents collisions when multiple servers expose same-named tools (e.g. filesystem__read_file).
STEP 04
Claude decides which tools to call
The task + all tool schemas are sent to Claude. Claude reasons over the task and emits tool_use blocks specifying which tool to call and with what arguments.
STEP 05
Execute tool calls via MCP server
Each tool_use block is routed to the correct MCP server's call_tool() method. Results (text content blocks) are collected and returned as tool_result messages.
STEP 06
Iterative loop until task complete
Tool results are appended to the message history and sent back to Claude. Claude continues calling tools or producing a final end_turn response. Hard cap at 20 iterations to prevent runaway loops.
💡 Use Cases
Explore a codebase: list components, read files, summarise architecture Query a database via MCP SQL server without writing SQL manually Run Git operations (log, diff, branch) through Claude's reasoning Connect to any third-party MCP server (Stripe, GitHub, Slack, Notion…) Chain multiple MCP servers in a single agentic session
🏹 Output
Tool call trace (console) Claude final answer Token + cost summary
